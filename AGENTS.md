# AGENTS.md

## Build
- Install: `.\pess-install.ps1` (once) or `bash pess-install.sh`
- Init project: `pess-init -ProjectName "x" -ProjectType "y"` or `bash pess-init.sh`
- Test: `pytest tests/` or `npm test`
- Lint: `ruff check hooks/` (self-check only)

## Constraints

### MUST and MUST NOT
- Hook scripts MUST stay in `hooks/` (source); deployment via pess-install.ps1
- Template files MUST NOT contain hardcoded absolute paths
- CLAUDE.md content MUST NOT be auto-generated by LLM (ETH Zurich arXiv:2602.11988)
- Skill description fields MUST contain ONLY triggering conditions, not workflow summaries
- Never push directly to main branch (MUST use PR)

### SHOULD and SHOULD NOT
- Memory Bank files SHOULD be updated at /checkpoint and /wrap
- Lessons SHOULD be recorded when a real bug is discovered and fixed
- You SHOULD read memory-bank/lessons.md before making similar mistakes

### MAY
- Agents MAY deviate from Constitution only if user explicitly requests it
- You MAY request clarification before generating code if requirements are ambiguous

## Architecture
- Entry point: pess-init.ps1 / pess-init.sh (project scaffolding)
- Global install: pess-install.ps1 / pess-install.sh (one-time setup)
- Templates: templates/ (read-only source of truth)
- Hooks: hooks/ (source); deployed to ~/.claude/hooks/ by installer

## File Roles
- hooks/*.py          → user-level security guards (NOT repo-level)
- templates/global-CLAUDE.md → copied to ~/.claude/CLAUDE.md by installer
- templates/commands/  → copied into .claude/commands/ by pess-init
- templates/skills/    → copied into .claude/skills/ by pess-init
- templates/*/CLAUDE.md → project-type-specific templates
- templates/adr-template.md → ADR writing template

## Multi-Tool Support
PESS targets Claude Code as primary. When setting up for other tools:
- Claude Code: .claude/commands/ + .claude/skills/ + CLAUDE.md
- Cursor: .cursorrules (run `pess-init --tool cursor`)
- GitHub Copilot: .github/copilot-instructions.md (run `pess-init --tool copilot`)
