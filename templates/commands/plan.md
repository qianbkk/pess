# /plan — 任务分解与执行计划

**第一步：读取上下文（只读与本次任务直接相关的文件）**
1. 读 memory-bank/activeContext.md（当前状态）
2. 如果涉及架构变化，读 memory-bank/systemPatterns.md
3. 如果是 Bug 修复，读 memory-bank/lessons.md

**第二步：生成执行计划**
格式：
```

影响文件：
  修改：[文件路径列表]
  新建：[文件路径列表]
  不影响：[容易误改的文件，明确排除]

步骤：
1. [具体操作，精确到函数/类级别]
2. [...]
...

测试验证：
- 单元测试：[哪些函数需要测试]
- 手动验证：[操作步骤]

预计影响范围：[小/中/大]
```

**第三步：等待确认**
输出计划后等待我说"ok"、"开始"或具体修改意见，不要自动开始执行。
