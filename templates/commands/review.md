# /review — 多维度代码审查

对 `git diff HEAD` 或指定的文件执行以下检查：

**Round 1：正确性**
- [ ] 有无未处理的 None/null/undefined 情况？
- [ ] 错误处理是否完整（无裸 except/catch）？
- [ ] 有无资源泄漏（文件、连接、锁未关闭）？
- [ ] 并发场景是否安全（共享状态、race condition）？

**Round 2：规范合规**
- [ ] 是否符合 CLAUDE.md 的所有硬规则？
- [ ] 命名是否与项目现有代码风格一致？
- [ ] 是否违反了 memory-bank/systemPatterns.md 中的架构模式？
- [ ] 是否有魔法数字？是否有注释解释非显而易见的逻辑？

**Round 3：安全（涉及 auth/input/DB/API 时必做）**
- [ ] 有无 secret 硬编码？
- [ ] 用户输入是否经过验证？
- [ ] 数据库操作是否使用参数化查询？
- [ ] API 端点是否有认证/授权检查？
- [ ] 日志输出是否包含敏感数据？

**Round 4：测试完整性**
- [ ] 核心逻辑是否有对应测试？
- [ ] 是否覆盖了异常路径？
- [ ] 测试是否依赖硬编码数据（应用 Fixture）？

**输出格式：**
- ✅ 通过
- ⚠️ 建议改进（不阻断）：[说明]
- ❌ 必须修复（阻断）：[说明 + 建议修法]
