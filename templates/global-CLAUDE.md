# 全局工程规范

## 绝对禁令
- 禁止在我说"执行"之前修改任何文件
- 禁止一次修改超过3个文件，除非我明确说"批量操作"
- 禁止删除任何注释
- 禁止硬编码配置值（端口/路径/密钥/URL）
- 禁止对同一错误重试超过2次，第3次必须停下来描述根因
- 禁止生成超过150行的函数，必须先拆分
- 禁止在未先输出计划的情况下开始写代码

## 代码质量硬规则
- 所有函数必须有参数类型注解和返回类型注解
- 禁止裸 except/catch，必须指定异常类型
- 禁止魔法数字，必须定义命名常量
- 禁止"TODO: fix later"注释，要么立即修，要么记录为 Issue

## 安全硬规则（适用所有项目）
- 禁止在客户端代码中出现 secret/token/key 变量的赋值
- 所有接收用户输入的函数必须有显式输入验证
- 数据库查询禁止字符串拼接，必须用参数化查询
- 任何 HTTP 端点必须有认证检查（除非显式标注 public=true）

## 输出格式规则
- 收到任务：先输出编号计划（1/2/3...），等我说"ok"或"开始"
- 修改超过1个文件：先列出所有将修改的文件路径
- 遇到架构决策不确定：停下来问我，不要假设

## 环境约定
- OS: Windows 11，路径必须用 os.path.join 或 pathlib.Path
- 代理端口: 7890/7897（访问境外服务）
- Node.js: D:\AI\Node.js\（非标准路径，脚本中显式引用）
